-- Add-on Summary 
（从哪到哪）从哪到的一个网页，从那些网页又访问了哪些网页。0.09起支持关键词查询，0.19加强；0.12起Scrapbook支持；0.14起简陋“共享”支持。

-- Add-on Description 
<b>日常用例：</b>

1） 一边搜索一边开网页，还想继续搜索的时候发现原本的搜索页面找不到了，可能错手关了，或者被新页面覆盖。历史记录看得到有一系列“某某 - 搜索”，但我只想知道当前的页面是从是哪个搜索页面打开的。我可以挨个试，也可以打开FromWhereToWhere追溯源头，然后打开。<b>请注意</b>：据个人测试火狐更新浏览历史可能需2分钟。

2） 几个月前为解决一个Linux配置问题花一个小时搜索加尝试，结果最后搞定欢呼之后忘了留下记录。今天又在另个机器“幸运”碰到同样问题，第一反应是开搜，说不定碰运气和记忆能找到那个解决方案。眼熟的网页不少，但没一个是“正解”。从眼熟的网页至少能用FromWhereToWhere列出之前都访问过哪些相关网页（法同1）。

3） 不用非得找到一个已经访问过的网页，可以直接以关键词搜索网页的题目（非全文），同时列出相关网页（从哪来，又去哪）。包含关键词的题目高亮显蓝。

4） <b>共享</b> 朋友问问题，马上变身“搜索机器”。几分钟之后找到些可能链接，不过一般也会把关键词一同发给朋友，所谓授人以渔。现在不用把每个链接和关键词拷贝下来，只需在FromWhereToWhere视图中找到相关的节点，然后用“Property/Import”功能（见HOWTO“自0.14起”）与朋友共享（需安装同样插件）。

<b>提示：FromWhereToWhere<em>从不</em>记录浏览历史。火狐只在你允许时记录。FromWhereToWhere仅进行历史搜索，这也意味着可以在碰到上述类似问题时才安装此插件，便能查询迄今火狐保存的所有历史记录。</b>

<b>简介：</b>

与“Referrer History”类似，显示如何通过网页链接到达另外网页。差别在于，只显示当前网页相关，或者关键词相关，而非整个浏览历史。 

<b>HOWTO:</b>

用“工具 -> FromWhereToWhere”打开树视图，内容将基于当前网页。搜索仅返回包含所有关键词的题目。

树视图中，子节点为通过父节点中的链接到达的，极端情况可有上千，省略号表示查询进行中。可双击或右键打开某个节点的网页。

如果某个节点无子节点，代表未从它的链接打开其他网页。若没有网页题目包含所有关键词，将提示搜索无结果。

自0.12，若网页被Scrapbook抓取过，将斜体显示。右键菜单可打开本地备份。

自0.14，右键菜单添加“Property”与“Import”，可用于共享。选择需要共享的节点，右键选择“Property”，在打开的对话框中选择拷贝。<b> 注：</b>节点若不展开则不会包含子节点信息，是个检查有无“敏感”信息的机会。最后访问日期暂不包含在属性信息中。将拷贝下来的文本发给朋友，在他们的FromWhereToWhere视图中选择“Import”，将会原样显示树结构。

自0.19，支持带引号精确搜索。极端测试为空引号“”，将列出所有历史记录（耗时自不用说）。减号可排除关键词。
